## 💡 经验总结

### 技术经验
1. **Next.js 动态路由** - 使用 `export const dynamic = 'force-dynamic'` 明确声明
2. **React Query 缓存** - 合理设置 `staleTime` 和 `gcTime` 平衡性能和新鲜度
3. **Supabase 数组查询** - 使用 `.filter('cs')` 操作符
4. **数据库索引** - GIN 索引适合数组字段，B-Tree 适合精确匹配

### 协作经验
1. **批量更新数据** - 先 `--dry-run` 预览，确认后再 `--execute`
2. **分类标准化** - 参考学术标准（CVPR/IGARSS），避免主观分类
3. **提交纪律** - 不要频繁提交，等用户确认后再推送

### 设计经验
1. **导航一致性** - 首页和搜索页保持统一风格
2. **可点击区域** - 仅文字可点击，避免整行误触
3. **内容去重** - 避免同一功能多处出现（如论文链接）
