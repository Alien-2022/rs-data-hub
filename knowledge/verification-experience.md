## 🏆 核心经验（2026-03-27 验证 20 个数据集总结）

### 1. 论文链接验证优先级

**✅ 最佳实践**: arxiv 优先

| 优先级 | 平台 | 成功率 | 说明 |
|--------|------|--------|------|
| 1 | **arxiv** | 90%+ | 开源预印本，访问稳定，强烈推荐 |
| 2 | **期刊官网** | 80% | 如遥感学报，作为引导页面足够 |
| 3 | **IEEE/ScienceDirect** | 60% | 有付费墙，但通常有 arxiv 替代 |
| 4 | **个人项目页面** | 50% | 可能失效，需定期验证 |

**验证结果** (2026-03-27):
- 20 个数据集中 9 个找到 arxiv 链接 (45%)
- 所有 arxiv 链接验证成功 (100% 可用)
- IEEE 付费墙论文均可在 arxiv 找到替代

## 🛠️ 验证工具与命令

### 浏览器验证
最直接的方式，适合少量验证：
```
1. 打开浏览器
2. 访问链接
3. 检查页面是否正常加载
4. 截图记录（可选）
```

### 命令行验证 (curl)
适合批量验证：
```bash
# 检查 HTTP 状态码
curl -o /dev/null -s -w "%{http_code}\n" https://arxiv.org/abs/1909.00133
# 输出：200

# 获取页面标题
curl -s https://arxiv.org/abs/1909.00133 | grep "<title>"
```

### web_fetch 工具
适合获取页面内容验证：
```bash
# 提取页面文本
web_fetch --url https://arxiv.org/abs/1909.00133 --extract-mode text
```

## 💡 实用技巧

### 技巧 1: 快速查找 arxiv 论文
**搜索格式**: `[数据集名称] + arxiv`

**示例**:
```
DIOR remote sensing arxiv
MLRSNet deep learning arxiv
FAIR1M fine-grained arxiv
```

**搜索引擎**:
- Google: `site:arxiv.org [数据集名称]`
- arxiv 站内搜索: https://arxiv.org/search/

---

### 技巧 2: 处理付费墙论文
**问题**: IEEE/ScienceDirect/Springer 论文需要机构访问

**解决方案**:
1. 搜索 arxiv 预印本（成功率 80%+）
2. 搜索作者个人页面（通常有 PDF 下载）
3. 使用 Sci-Hub（不推荐，版权风险）
4. 邮件联系作者索取

**实际案例**:
- MLRSNet: ScienceDirect 付费 → 找到 arxiv:2010.00243
- FAIR1M: ScienceDirect 付费 → 找到 arxiv:2103.05569

---

### 技巧 3: 找替代下载源
**问题**: 官方下载链接失效

**解决方案**:
1. 搜索 `数据集名称 + Kaggle`
2. 搜索 `数据集名称 + GitHub`
3. 搜索 `数据集名称 + download`
4. 邮件联系作者

**实际案例**:
- SSDD: 旧 GitHub 404 → 找到官方新仓库 Official-SSDD
- HRSC2016: 西工大页面 → Kaggle 镜像

---

### 技巧 4: 验证 GitHub 仓库活跃度
**检查项**:
- [ ] 最后更新时间（1 年内为佳）
- [ ] Star 数量（>50 较可靠）
- [ ] Issue 是否有人回复
- [ ] README 是否完整

**示例**:
```
✅ TianwenZhang0825/Official-SSDD
   - 最后更新：2025 年
   - Star: 200+
   - README 完整
   
⚠️ 某个人仓库
   - 最后更新：3 年前
   - Star: 5
   - 链接 404
```

---

### 技巧 5: 验证期刊官网作为引导页面
**原则**: 不一定要能直接下载 PDF，能作为论文引导页面就足够

**检查项**:
- [ ] 页面包含论文标题
- [ ] 页面包含作者信息
- [ ] 页面包含 DOI
- [ ] 页面提供引用格式

**实际案例**:
- MAR20: 遥感学报官网，虽不能直接下载，但作为论文页面完全合格
- LEVIR-CD: 个人项目页面，包含完整数据集描述和引用信息

## ⚠️ 常见陷阱与解决方案

### 陷阱 1: web_fetch 访问失败 ≠ 链接失效
**问题**: 使用 web_fetch 工具访问失败，误判为链接失效

**原因**:
- 网站有反爬虫机制（ScienceDirect、IEEE）
- 需要 JavaScript 渲染
- 需要登录

**解决方案**:
- 用浏览器手动验证
- 使用 curl 检查 HTTP 状态码
- 找替代平台（如 arxiv）

**实际案例**:
- DIOR: web_fetch 失败 → 浏览器访问 arxiv 成功
- MLRSNet: ScienceDirect 机器人验证 → 找到 arxiv 替代

---

### 陷阱 2: GitHub 404 不代表数据集消失
**问题**: GitHub 仓库 404，误判为数据集不可用

**原因**:
- 作者迁移仓库
- 仓库改名
- 账号封禁

**解决方案**:
- 搜索作者新账号
- 搜索数据集名称找镜像
- 邮件联系作者

**实际案例**:
- SSDD: `TianwenZhang0825/SSDD_Ship_Detection` 404 
  → 找到 `TianwenZhang0825/Official-SSDD`

---

### 陷阱 3: 大学官网链接容易失效
**问题**: 大学实验室官网改版导致链接失效

**原因**:
- 网站重构
- 服务器迁移
- 毕业生页面清理

**解决方案**:
- 优先找官方平台镜像（Kaggle、NODA 等）
- 搜索作者新单位页面
- 使用 Wayback Machine 找历史页面

**实际案例**:
- WHU-CD: 武大遥感官网 → 找到 GPCV 新页面
