# 数据集分类标准化说明

**更新时间**: 2026-03-29  
**目的**: 统一和简化遥感数据集的任务类型和数据模态分类，提升用户体验和搜索准确性

---

## 📊 变更原因

### 问题 1：分类重叠

**原分类**（14 个任务类型）：
```
图像分类、场景分类、土地利用分类、土地覆盖分类、
目标检测、旋转目标检测、细粒度检测、
语义分割、实例分割、变化检测、
多标签分类、细粒度分类
```

**问题**：
- "场景分类"、"土地利用分类"、"土地覆盖分类" 本质上都是图像分类
- "旋转目标检测"、"细粒度检测" 是目标检测的子类
- "多标签分类" 是任务属性，不是任务类型

### 问题 2：维度混乱

**原分类**（9 个数据模态）：
```
光学、SAR、多光谱、
航拍、无人机、无人机航拍、
光学/航拍、航拍 (IR-R-G)、航拍 (RGB+NIR)
```

**问题**：
- "航拍"、"无人机" 是数据采集平台，不是传感器模态
- "光学/航拍" 混合了传感器和平台两个维度
- 分类过于具体，不利于筛选

---

## ✅ 标准化方案

### 任务类型（14 → 5）

| 新分类 | 包含的旧分类 | 说明 |
|--------|--------------|------|
| **图像分类** | 图像分类、场景分类、土地利用分类、土地覆盖分类、细粒度分类、多标签分类 | 整图级别的分类任务 |
| **目标检测** | 目标检测、旋转目标检测、细粒度检测 | 定位 + 识别目标 |
| **语义分割** | 语义分割、土地覆盖分类 | 像素级分类 |
| **变化检测** | 变化检测 | 双时相分析 |
| **实例分割** | 实例分割 | 实例级分割 |

### 数据模态（9 → 5）

| 新分类 | 包含的旧分类 | 说明 |
|--------|--------------|------|
| **光学** | 光学、光学/航拍、航拍 | 可见光/红外传感器 |
| **SAR** | SAR | 合成孔径雷达 |
| **多光谱** | 多光谱、无人机、无人机航拍、航拍 (IR-R-G)、航拍 (RGB+NIR) | 多波段传感器 |
| **高光谱** | 高光谱 | 高光谱分辨率 |
| **LiDAR** | LiDAR | 激光雷达 |

## 📝 数据集变更详情

### 变更统计

| 数据集 | 任务类型变更 | 数据模态变更 |
|--------|--------------|--------------|
| NWPU-RESISC45 | 场景分类，图像分类 → **图像分类** | 无变化 |
| DOTA v2.0 | 目标检测，旋转目标检测 → **目标检测** | 光学/航拍 → **光学** |
| EuroSAT | 场景分类，土地利用分类 → **图像分类** | 无变化 |
| BigEarthNet | 场景分类，土地利用分类，多标签分类 → **图像分类** | 无变化 |
| UAVid | 无变化 | 无人机航拍 → **多光谱** |
| OpenEarthMap | 语义分割，土地覆盖分类 → **语义分割** | 航拍 → **光学** |
| AID | 场景分类 → **图像分类** | 航拍 → **光学** |
| MLRSNet | 场景分类，多标签分类 → **图像分类** | 光学/航拍 → **光学** |
| FAIR1M | 细粒度检测，目标检测 → **目标检测** | 无变化 |
| 其他 11 个数据集 | 简化或保持不变 | 简化或保持不变 |

**总计**: 20 个数据集中，约 10-12 个需要更新

## 📈 预期效果

### 用户体验提升

| 方面 | 优化前 | 优化后 |
|------|--------|--------|
| **筛选栏长度** | 14 个任务类型，9 个模态 | 5 个任务类型，5 个模态 |
| **选择难度** | 用户难以区分细分类别 | 清晰的主类，容易理解 |
| **搜索结果** | 可能遗漏相关数据集 | 更全面的召回 |
| **专业性** | 分类标准不统一 | 符合学术标准 |

### 技术优势

- ✅ 分类体系清晰，易于维护
- ✅ 符合遥感领域标准（CVPR/IGARSS）
- ✅ 便于未来扩展（可添加子分类）
- ✅ 减少数据库存储（去重）

## 🎯 未来扩展

### 可选：添加子分类

如果未来需要更细粒度的筛选，可以添加子分类标签：

```
图像分类
├── 场景分类
├── 土地利用分类
└── 土地覆盖分类

目标检测
├── 通用目标检测
├── 旋转目标检测
└── 细粒度检测
```

**实现方式**：
- 主分类：`task_types` 字段（用于筛选）
- 子分类：`tags` 字段（用于标签云）

## ⚠️ 注意事项

### 数据备份

执行更新前，建议备份现有数据：

```bash
# 导出当前数据
python export_datasets.py --output datasets_backup_$(date +%Y%m%d).json
```

### 回滚方案

如果更新后发现问题，可以回滚：

```bash
# 使用备份文件恢复
python import_datasets.py --input datasets_backup_YYYYMMDD.json --mode replace
```

### 缓存清除

更新后，建议：
1. 清除 Vercel 缓存（重新部署）
2. 清除浏览器缓存
3. 测试搜索和筛选功能
